Data Sources

Data on Déjà View comes from multiple sources. Each source has unique characteristics.

Billboard publishes the Billboard Hot 100 list at www.billboard.com/charts/hot-100. There is no API so we had to parse the HTML. Here's how we got what we needed for a given week:
  1. Send a request to the Billboard Hot 100 page.
  2. Store the artist, song title, and number of weeks on the chart for the top 10 songs in a JSON file as a backup.
  3. Lookup the video on YouTube.
The Numbers publishes popular movies' gross box office sales at www.the-numbers.com/box-office. There is no API so we had to parse the HTML. Here's how we got what we needed for a given week:
  1. Send a request to The Numbers page.
    • See if there is a listing for a given week. If so, use the weekly chart.
    • If not, see if there is a listing for the weekend.
  2. Store the movie, and gross sales for all available movies in a JSON file as a backup.
  3. Lookup the trailer on YouTube.
The New York Times provides a public API at developer.nytimes.com. Here's how we got what we needed for a given date:
  1. Send a request to the Article Search API.
  2. Store the raw data in a JSON file as a backup.
  3. Use only the headline, link, snippet, and multimedia in our database.
Wikipedia lists important events for every calendar day at links similar to en.wikipedia.org/wiki/April_1. We had to parse the HTML. Here's how we got what we needed for a given date:
  1. Send a request to the Wikipedia page.
  2. Store the raw data in a JSON file as a backup.
  3. Use only the events and births in our database.

Staying Up-to-Date

In order to maintain current data, we scheduled an AWS Lambda function to run periodically to retrieve new data from the sources that are updated regularly.

You can read more about it at github.com/xzjia/dejaview-scraper.
